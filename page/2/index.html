为github page绑定域名

发表于 2018-07-02 更新于 2019-10-16

为github page绑定域名

配置github page后可以为通过CNAME为github page绑定一个自己的域名。

购买域名

  • 在域名服务上购买域名
  • 实名认证域名
    • 可以在阿里云、华为云等通过上传身份证快速通过实名认证

设置域名解析

  • 阿里云域名提供解析功能

    • 在这里找到注册域名。
    • 配置解析规则
      记录类型 主机记录 解析线路 记录值 MX优先级 TTL
      CNAME * 境外 yourname.github.io – 10
      CNAME www 默认 yourname.github.io – 10

  • 在github.com配置解析功能

    1. 在github中增加一个CNAME文件，文件内容

      www.yourdomain.name

      这个中方式有个问题，每次更新github中的CNAME文件都会被删除。域名访问测试成功后可以按照如下方式配置保留CNAME文件

    2. 在hexo soure目录下新增一个上述的CNAME文件，则每次都会把CNAME文件原样生成到github上

使用hexo和github构建静态博客网站

发表于 2018-07-01 更新于 2019-10-16

使用hexo和github构建静态博客网站

安装软件

  1. 安装Node.js
  2. 安装git工具
  3. 安装hexo工具
    npm i -g hexo

配置github

  1. 注册github账号
  2. 创建github repo
    创建一个repo，名称为yourname.github.io, 其中yourname是你的github名称，按照这个规则创建才有用
  3. 配置github ssh key
    • 在C:/user/username 目录下新建。ssh目录
    • 进入到 .ssh目录，执行ssh-keygen -t rsa -C “youremail@example.com“，生成ssh
    • 在.ssh目录中找到id_rsa.pub文件的内容，并将内容拷贝到github里面
    • 通过ssh -T git@github.com检查配置的key是否生效

使用hexo创建博客

  1. 配置根目录下_config.yaml文件
    1). type:git
    2). repo:git@github.com:youname/youname.github.io.git
    3). branch:master
  2. 通过hexo d 部署博文到github

通过hexo发布表新文章

发表于 2018-07-01 更新于 2019-10-16

通过hexo更新文章

通过hexo n “文档标题”

  1. hexo n 创建一个新博客，该命令在source/_post/目录下生成一个“文档标题”.md文档
  2. 编辑新建的md文档即可生成新博客

通过hexo g 重新生成网页

重新生成静态文件

通过hexo s 重启本地服务器

通过hexo s启动本地服务器，访问本地服务器 http://localhost:4000，即可预览博文。

发布到github

执行hexo d

  1. 配置根目录下_config.yaml文件
    1). type:git
    2). repo:git@github.com:youname/youname.github.io.git
    3). branch:master
  2. 配置通过ssh key访问 git

更新主题

  1. 在blog执行git clone命令下载最新的主题
  2. 在theme目录下找到最新的主题目录，修改_config.yaml文件配置主题
  3. hexo clean清理
  4. hexo g 重新生成blog
  5. hexo s 启动本地服务器，预览博客
  6. hexo d 部署博客到github
